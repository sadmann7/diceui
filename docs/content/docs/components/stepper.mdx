## Layout

Import the parts, and compose them together.

```tsx
import {
  Stepper,
  StepperContent,
  StepperItemDescription,
  StepperItemIndicator,
  StepperItem,
  StepperList,
  StepperItemSeparator,
  StepperItemTitle,
  StepperItemTrigger,
} from "@/components/ui/stepper";

<Stepper defaultValue="step-1">
  <StepperList>
    <StepperItem value="step-1">
      <StepperItemTrigger value="step-1">
        <StepperItemIndicator value="step-1">1</StepperItemIndicator>
      </StepperItemTrigger>
      <StepperItemTitle>Step 1</StepperItemTitle>
      <StepperItemDescription>First step description</StepperItemDescription>
      <StepperItemSeparator />
    </StepperItem>
    <StepperItem value="step-2">
      <StepperItemTrigger value="step-2">
        <StepperItemIndicator value="step-2">2</StepperItemIndicator>
      </StepperItemTrigger>
      <StepperItemTitle>Step 2</StepperItemTitle>
      <StepperItemDescription>Second step description</StepperItemDescription>
      <StepperItemSeparator />
    </StepperItem>
    <StepperItem value="step-3">
      <StepperItemTrigger value="step-3">
        <StepperItemIndicator value="step-3">3</StepperItemIndicator>
      </StepperItemTrigger>
      <StepperItemTitle>Step 3</StepperItemTitle>
      <StepperItemDescription>Final step description</StepperItemDescription>
    </StepperItem>
  </StepperList>

  <StepperContent value="step-1">Content for step 1</StepperContent>
  <StepperContent value="step-2">Content for step 2</StepperContent>
  <StepperContent value="step-3">Content for step 3</StepperContent>
</Stepper>
```

## Usage Guidelines

### Step Values

The stepper uses string values to identify steps, following Radix UI patterns:

- Each step must have a unique `value` prop (e.g., "step-1", "personal-info", "checkout")
- Use descriptive values that make sense in your application context
- The stepper's `value`/`defaultValue` should match one of the step values

### Step States

The stepper automatically manages three states for each step through the store:

- **Inactive**: Steps that haven't been reached yet
- **Active**: The current step the user is on (matches the stepper's value)
- **Completed**: Steps marked as completed through the store's state management

### Navigation

- By default, steps are clickable and users can navigate between them
- Use the `clickable={false}` prop to prevent clicking on steps
- Individual steps can be disabled using the `disabled` prop on `StepperItem`

### Orientation

- **Horizontal**: Default layout, steps flow left to right
- **Vertical**: Compact layout, steps flow top to bottom

### Form Integration

When using with forms:

1. Validate each step before allowing progression
2. Use controlled state to manage the current step value
3. Implement proper error handling and validation feedback
4. Consider saving progress between steps for better UX
