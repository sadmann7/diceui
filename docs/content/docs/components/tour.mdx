## Layout

Import the parts, and compose them together.

```tsx
import * as Tour from "@/components/ui/tour";

<Tour.Root>
  <Tour.Backdrop />
  <Tour.Step target="#element-id">
    <Tour.Content>
      <Tour.Header>
        <Tour.Title />
        <Tour.Description />
      </Tour.Header>
      <Tour.Footer>
        <Tour.StepCounter />
        <Tour.Navigation>
          <Tour.PrevButton />
          <Tour.NextButton />
        </Tour.Navigation>
        <Tour.SkipButton />
      </Tour.Footer>
      <Tour.CloseButton />
    </Tour.Content>
  </Tour.Step>
</Tour.Root>
```

### ARIA Attributes

The tour component implements proper ARIA attributes for accessibility:

- `aria-label` attributes on navigation buttons for screen readers
- `role="dialog"` on the tour content for proper dialog semantics
- Focus management to ensure keyboard navigation works correctly
- Proper heading hierarchy with the title element

### Focus Management

- When a tour step becomes active, focus is managed appropriately
- The tour content receives focus when opened
- Focus is trapped within the tour content during navigation
- Focus returns to the trigger element when the tour is closed

## Notes

- The tour automatically scrolls to target elements when `scrollToElement` is enabled
- Target elements can be specified using CSS selectors, element references, or direct element objects
- The backdrop creates a visual highlight around the target element using CSS clip-path
- Tour steps are registered dynamically and can be added or removed at runtime
- The component supports both controlled and uncontrolled usage patterns
